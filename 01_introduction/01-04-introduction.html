L'ordre d'exécution par le navigateur

Pr défaut, un navigateur va lire et éxecuter le code dans l'ordre de son écriture

lorsque le navigateur arrive à un élément script, il va stopper le traitement du reste du HTML jusqu'à ce que le code Javascript soit chargé dans la page exécuté.

Résumons ce qu'il se passe dans ce cas

  1. Le navigateur commence à analyser ( ou à traiter) le code HTML
  2. Le navigateur rencontre un élément script
  3. Le contenu JS est demandé et téléchargé( si il est en externe) puis éxécuté Durant tout e temps, l'analyseur du navigateur bloque l'affichage du HTML, ce qui peujt ralentir, si le script est long, l'affichage de la page
  4. Dès que le JS a été exécuté, le contenu HTML est affiché

Ce souci d'attente est résolu grâce au ytéléchargement asynchrome des données avec l'attribut async ou defer dans le cas ou l'on fait appel a des fichcier extenres

Le ppoint virgule

Le point virgule ; est utilisé pour signifier la fin d'une instruction ( une variable, une condition, une boucle etc...)
